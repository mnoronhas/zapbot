       - Import `createBrowserClient` from `@supabase/ssr`

         ```
           process.env.NEXT_PUBLIC_SUPABASE_URL!,
           process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
         )
         ```

       - Import `createServerClient` from `@supabase/ssr`

         - Get cookieStore via `await cookies()`
         - Return `createServerClient(SUPABASE_URL, SUPABASE_ANON_KEY, { cookies: { getAll, setAll } })`

       - Follow Pattern 7 from research exactly.

       - Import `createServerClient` from `@supabase/ssr`
       - Import `NextResponse`, `NextRequest` from `next/server`
       - Create supabase client with request cookies (getAll from request.cookies, setAll on response cookies)
       - CRITICAL: Use `supabase.auth.getUser()` NOT `getSession()` -- getSession does NOT verify the JWT server-side
       - If no user AND path is not /login, /register, /auth/* -> redirect to /login
       - Export config with matcher that excludes static files: `["/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)"]`

       - Root layout with html lang="pt-BR"
       - Minimal metadata: title "ZapBot", description "Construtor de chatbots para WhatsApp"
       - Simple body wrapper, no Tailwind needed

    8. Create `apps/web/src/app/page.tsx`:
       - Server Component
       - Import server Supabase client
       - Check if user is logged in via `getUser()`
       - If yes, redirect to /dashboard
       - If no, redirect to /login

    9. Create `apps/web/src/app/auth/callback/route.ts`:
       - Handle GET requests
       - Extract `code` from URL search params
       - If code exists, create server Supabase client and call `exchangeCodeForSession(code)`
       - Redirect to /dashboard on success (or /login with error param on failure)
       - This handles both Google OAuth callback and email confirmation link callback

    10. Run `cd zapbot-project/zapbot && pnpm install` to install dependencies and generate lockfile.
