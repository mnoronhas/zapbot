<task type="auto">
  <name>Task 1: Add RLS policies to all tenant-scoped tables</name>
  <files>
    zapbot-project/zapbot/packages/db/src/schema/index.ts
  </files>
  <action>
    Modify the existing `packages/db/src/schema/index.ts` to add RLS policies to ALL tenant-scoped tables.

    1. Add new imports at the top of the file:
       ```
       import { pgPolicy } from "drizzle-orm/pg-core";
       import { authenticatedRole, supabaseAuthAdminRole } from "drizzle-orm/supabase";
       import { sql } from "drizzle-orm";
       ```

    2. Define a reusable SQL helper BEFORE the table definitions:
       ```typescript
       // Helper: resolves current authenticated user's account_id
       const myAccountId = sql`(
         SELECT id FROM accounts
         WHERE supabase_user_id = auth.uid()
       )`;
       ```

    3. Modify the `accounts` table definition to add RLS:
       - Add `.enableRLS()` to the table (check if drizzle-orm 0.33.0 supports it on pgTable; if not, we'll handle it in the migration)
       - Add pgPolicy in the third argument array (alongside existing indexes):
         ```
         pgPolicy("accounts_tenant_isolation", {
           for: "all",
           to: authenticatedRole,
           using: sql`supabase_user_id = auth.uid()`,
           withCheck: sql`supabase_user_id = auth.uid()`,
         })
         ```
       - Keep all existing indexes unchanged.

    4. Modify ALL other tenant-scoped tables to add RLS policies.
       Each table that has `account_id` gets the SAME policy pattern:
       ```
       pgPolicy("{table_name}_tenant_isolation", {
         for: "all",
         to: authenticatedRole,
         using: sql`account_id = ${myAccountId}`,
         withCheck: sql`account_id = ${myAccountId}`,
       })
       ```

       Tables to modify (all have account_id):
       - `whatsappConnections` -> policy name: "wa_connections_tenant_isolation"
       - `bots` -> policy name: "bots_tenant_isolation"
       - `calendarConfigs` -> policy name: "calendar_configs_tenant_isolation"
       - `conversations` -> policy name: "conversations_tenant_isolation"
       - `appointments` -> policy name: "appointments_tenant_isolation"
       - `analyticsEvents` -> policy name: "analytics_events_tenant_isolation"

    5. Tables WITHOUT account_id directly but with conversation_id:
       - `messages` -> needs a policy that joins through conversations to get account_id:
         ```
         pgPolicy("messages_tenant_isolation", {
           for: "all",
           to: authenticatedRole,
           using: sql`conversation_id IN (
             SELECT id FROM conversations
             WHERE account_id = ${myAccountId}
           )`,
           withCheck: sql`conversation_id IN (
             SELECT id FROM conversations
             WHERE account_id = ${myAccountId}
           )`,
         })
         ```
       - `botVersions` -> needs a policy through bots:
         ```
         pgPolicy("bot_versions_tenant_isolation", {
           for: "all",
           to: authenticatedRole,
           using: sql`bot_id IN (
             SELECT id FROM bots
             WHERE account_id = ${myAccountId}
           )`,
           withCheck: sql`bot_id IN (
             SELECT id FROM bots
             WHERE account_id = ${myAccountId}
           )`,
         })
         ```

    6. Add `.enableRLS()` to every table definition. In drizzle-orm, this is typically chained on the pgTable call. Check the API:
       - If `pgTable(...).enableRLS()` is supported, use it
       - If not, the migration will need manual `ALTER TABLE ... ENABLE ROW LEVEL SECURITY;` statements -- note this in a comment at the top of the schema file

    CRITICAL NOTES:
    - Keep ALL existing column definitions exactly as they are. Do NOT change column types, defaults, or constraints.
    - Keep ALL existing indexes. Add pgPolicy calls alongside them in the third argument array.
    - The `for: "all"` covers SELECT, INSERT, UPDATE, DELETE operations in one policy.
    - The policy names must be unique across the database.
  </action>

    - `grep -c "enableRLS" packages/db/src/schema/index.ts` returns 9 (one per table, or a count matching tables)
    - `grep "authenticatedRole" packages/db/src/schema/index.ts` returns matches

    - TypeScript compiles: `cd zapbot-project/zapbot && npx tsc --noEmit -p packages/db/tsconfig.json`
  </verify>
