       - Create RLS connection (separate postgres instance, also with `prepare: false`)
       - Create internal `rlsDb = drizzle(rlsConnection, { schema })`

         - `admin`: the adminDb (for non-RLS queries like webhook processing)
         - `rls`: a function that wraps `rlsDb.transaction()` and sets the JWT context via:
           ```
           SELECT set_config('request.jwt.claims', '<claims JSON>', TRUE);
           SELECT set_config('request.jwt.claim.sub', '<claims.sub>', TRUE);
           SET LOCAL ROLE authenticated;
           ```
           Then executes the user's transaction callback, and in a `finally` block resets:
           ```
           SELECT set_config('request.jwt.claims', NULL, TRUE);
           SELECT set_config('request.jwt.claim.sub', NULL, TRUE);
           RESET ROLE;
           ```

       Follow the pattern from the research (Pattern 2) exactly. Use `sql.raw()` for interpolating the claims JSON string. Use `sql` template literal for the SET commands.

    1. Create `packages/db/src/crypto.ts`:
       - Import `crypto` from `node:crypto`
       - Constants: `ALGORITHM = "aes-256-gcm"`, `IV_LENGTH = 12`, `AUTH_TAG_LENGTH = 16`

       - `encrypt(plaintext: string): string` function:
         - Get key via `getKey()`
         - Generate random IV: `crypto.randomBytes(IV_LENGTH)`
         - Create cipher: `crypto.createCipheriv(ALGORITHM, key, iv)`
         - Encrypt: `Buffer.concat([cipher.update(plaintext, "utf8"), cipher.final()])`
         - Get auth tag: `cipher.getAuthTag()`
         - Return `Buffer.concat([iv, authTag, encrypted]).toString("base64")`
         - Storage format: `base64(iv[12] + authTag[16] + ciphertext)`

       - `decrypt(encryptedBase64: string): string` function:
         - Get key via `getKey()`
         - Decode: `Buffer.from(encryptedBase64, "base64")`
         - Extract IV: `data.subarray(0, IV_LENGTH)`
         - Extract auth tag: `data.subarray(IV_LENGTH, IV_LENGTH + AUTH_TAG_LENGTH)`
         - Extract ciphertext: `data.subarray(IV_LENGTH + AUTH_TAG_LENGTH)`
         - Create decipher: `crypto.createDecipheriv(ALGORITHM, key, iv)`
         - Set auth tag: `decipher.setAuthTag(authTag)`
         - Decrypt: `Buffer.concat([decipher.update(ciphertext), decipher.final()]).toString("utf8")`

       - Private `getKey(): Buffer` function:
         - Read `process.env.ENCRYPTION_KEY`
         - Validate: must exist and be exactly 64 hex characters (32 bytes)
         - Throw clear error if missing or wrong length: "ENCRYPTION_KEY must be set and be 64 hex characters (32 bytes)"
         - Return `Buffer.from(keyHex, "hex")`
