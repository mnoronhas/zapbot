    1. Create `packages/db/src/trigger.sql`:
       This SQL must be run in the Supabase SQL Editor (it creates a trigger on auth.users which is a Supabase-managed table).

       ```sql
       -- ZapBot: Auto-create account row when a new user registers via Supabase Auth
       -- Run this in Supabase Dashboard -> SQL Editor

       CREATE OR REPLACE FUNCTION public.handle_new_user()
       RETURNS TRIGGER AS $$
       BEGIN
         INSERT INTO public.accounts (
           supabase_user_id,
           email,
           business_name
         ) VALUES (
           NEW.id,
           NEW.email,
           COALESCE(
             NEW.raw_user_meta_data->>'business_name',
             split_part(NEW.email, '@', 1)
           )
         );
         RETURN NEW;
       END;
       $$ LANGUAGE plpgsql SECURITY DEFINER;

       -- Drop existing trigger if re-running
       DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;

       CREATE TRIGGER on_auth_user_created
         AFTER INSERT ON auth.users
         FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
       ```

       CRITICAL: `SECURITY DEFINER` is required because the trigger function runs in the auth schema context but needs to write to public.accounts.
