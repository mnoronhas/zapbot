    1. Add dependencies to `apps/engine/package.json`:
       - `jose` (^5.0.0) -- JWT verification via JWKS
       - `lru-cache` (^10.0.0) -- Account lookup caching
       Run: `cd zapbot-project/zapbot && pnpm --filter engine add jose lru-cache`

    2. Create `apps/engine/src/middleware/auth.ts`:

       Imports:
       - `jwtVerify`, `createRemoteJWKSet` from `jose`
       - `onRequestHookHandler` type from `fastify`
       - `adminDb` from `@zapbot/db`
       - `accounts` from `@zapbot/db`
       - `eq` from `drizzle-orm`
       - `LRUCache` from `lru-cache`

       Constants:
       - `SUPABASE_URL = process.env.SUPABASE_URL!`
       - `JWKS = createRemoteJWKSet(new URL(\`${SUPABASE_URL}/auth/v1/.well-known/jwks.json\`))`
         This caches JWKS keys automatically (jose handles rotation).

       Account cache:
       - `accountCache = new LRUCache<string, typeof accounts.$inferSelect>({ max: 1000, ttl: 5 * 60 * 1000 })`
         5 minute TTL, max 1000 entries. Prevents N+1 DB queries per request.

       Fastify type augmentation (declare module):
       ```typescript
       declare module "fastify" {
         interface FastifyRequest {
           jwtClaims: Record<string, unknown>;
           account: typeof accounts.$inferSelect;
         }
       }
       ```

       Export `requireAuth: onRequestHookHandler`:
       1. Extract Bearer token from `request.headers.authorization`
          - If missing or not "Bearer ": return 401 with `{ error: "Token de autenticacao ausente", code: "MISSING_TOKEN" }`
       2. Verify token:
          ```
          const { payload } = await jwtVerify(token, JWKS, { audience: "authenticated" });
          ```
          - On failure: return 401 with `{ error: "Token invalido ou expirado", code: "INVALID_TOKEN" }`
       3. Attach claims to request: `request.jwtClaims = payload as Record<string, unknown>`
       4. Resolve account from `payload.sub` (Supabase user ID):
          - Check accountCache first
          - If miss, query: `adminDb.select().from(accounts).where(eq(accounts.supabaseUserId, supabaseUserId))`
          - If no account found: return 403 with `{ error: "Conta nao encontrada", code: "NO_ACCOUNT" }`
          - Cache the found account
       5. Attach account to request: `request.account = account`
       6. Wrap the entire verify+resolve in try/catch. Log warnings on failure: `request.log.warn({ err }, "JWT verification failed")`

       Error messages are in PT-BR per project convention. Error codes are in English for programmatic handling.

       NOTE on SUPABASE_URL: This MUST be set when the engine starts. If missing, createRemoteJWKSet will fail. Add a startup validation check or let it fail fast with a clear error.
  </action>
